Animations

Built independently or in combination with SVG, CSS3, JavaScript and the main ingredient: Velocity.js. The problem with most animations is the way its organized and executed. Most use jQuery as it provides the path of least resistance, Unfortunately it has incredibly poor performance. And while JavaScript animations are incredibly fast, implementing them is no small task. Velocity is its baby. Combining the straightforward interface of jQuery with the speed of JS, it will look and perform brilliantly anywhere. It also is fully compatible as far back as ie8, so it will work on any device built in this millennium.

Icons/graphics

There are 3 categories, regular, vector & SVG (Scalable Vector Graphics). Regular graphics are exactly what they sound like, regular. Including but not limited to .gif, .jpg, & .jpg. These are the default graphics used for the web. Straightforward to implement, but somewhat heavy ( ~ 50kb ) and likely to look grainy when observed on very high resolution monitors.

Vector Graphics are outgrowing regular graphics to become the baseline. This is because every line used is clean and crisp ensuring it looks good on just about every monitor it is displayed on. It is more difficult to create but well worth it.

SVG is the crème de la crème, undoubtedly the best way to provide graphics alas the most difficult to create. SVG is special for 2 reasons: pure code & infinite scalability. Being built from code means there are NO HTTP REQUESTS, it loads instantly and can take up as little as 5kb vs +25 kb for vector based. Infinite scalability means that the graphic will stay clean and crisp forever no mater how advanced displays get. Words do this no justice.
